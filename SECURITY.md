# Security Policy

Thank you for helping keep this project secure. Security reports are taken seriously, and we appreciate responsible disclosure from users, developers, researchers, and maintainers.

This document explains which versions are currently supported, how to report a vulnerability, what information to include, and what you can expect after submitting a report.

---

## Supported Versions

Security updates are only provided for actively supported versions of the project. Unsupported versions may still work, but they will not receive security patches unless explicitly stated.

| Version | Supported | Status |
| ------- | --------- | ------ |
| 5.1.x   | ✅ Yes | Current stable release. Security fixes and critical patches are provided. |
| 5.0.x   | ❌ No | No longer supported. Users should upgrade to 5.1.x or later. |
| 4.0.x   | ✅ Yes | Long-term support branch. Critical security fixes may be backported. |
| < 4.0   | ❌ No | Unsupported. Upgrade strongly recommended. |

### Recommendation

Users should always run the latest supported version where possible. Older unsupported versions may contain known vulnerabilities, outdated dependencies, or insecure behaviour that will not be fixed.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

Please do **not** publicly disclose the issue until it has been reviewed and a fix or mitigation has been prepared.

### How to Report

Report vulnerabilities using one of the following methods:

- Open a private GitHub Security Advisory, if available.
