Building a stronger online identity for kai9987kai.

This first post sets the direction for the blog: a place to collect experiments, explain project decisions, show progress, and turn scattered creative technology work into a clearer portfolio story.

✍️ Kai Piper 📅 09 Jun 2026 ⏱️ 4 min read ✨ Portfolio
In this post Why the blog exists The kai9987kai identity A better project system What comes next

The kai9987kai site should not feel like a folder of random links. It should feel like a live workshop: part portfolio, part dev log, part design notebook, and part experimental lab. Every project can show what it does, why it exists, what changed, and what could be improved next.

That is the reason for starting this blog properly. The aim is not just to post updates, but to build a clearer identity around creative coding, AI tools, 3D web experiments, AR prototypes, browser games, 3D printing utilities, and unusual design concepts.

The goal is simple: turn strange ideas into working pages, then explain the thinking behind them.

Why this blog exists

A GitHub repository can show the code, but it does not always show the journey. A portfolio page can show the finished result, but it can miss the experiments, failed attempts, UI improvements, bug fixes, and small decisions that made the project better.

This blog fills that gap. It gives each project a story: the original idea, what problem it solves, how the interface evolved, what features were added, and what the next version could become. That makes the work easier to understand for visitors, future collaborators, tutors, employers, and anyone looking through the site.

Blog rule:

Every post should leave the reader with a clear picture of the idea, the build, the result, and the next improvement.

The kai9987kai identity

The strongest direction for kai9987kai is experimental but usable. The ideas can be weird, playful, visual, or futuristic, but the final page should still be clear enough for someone to understand and interact with.

That means the identity should be built around a few repeating themes:

  • Creative technology: using code to make tools, visuals, games and prototypes.
  • Fast experiments: turning rough ideas into working demos instead of leaving them as notes.
  • Visual polish: making pages feel designed, not just functional.
  • Practical improvement: fixing bugs, improving controls, adding help, and making projects easier to use.

A better project system

To make the site feel more advanced, every blog post can use the same structure. This keeps the blog clean and makes the projects easier to compare.

1. The idea What sparked the project, and what should it let people do?
2. The build What tools, code, design choices or experiments were used?
3. The upgrade What was improved: UI, layout, speed, controls, accessibility or features?
4. The next version What could be added next to make the project more useful or impressive?

This structure works for AI tools, Three.js experiments, AR marker pages, browser games, slicer utilities, portfolio pages, and even one-off concept builds. It makes the blog feel organised while still leaving room for creativity. 

const postFormat = {
  idea: "What is the concept?",
  build: "How was it made?",
  upgrade: "What improved?",
  next: "Where does it go now?",
  identity: "experimental but usable"
};

What comes next

The next posts should focus on real kai9987kai projects. Good starting points would be AR Fire Lab, ThreeJSLab, the browser game experiments, 3D printing tools, AI interface ideas, and portfolio redesigns. Each one can become a mini case study instead of just a basic project link.

The long-term goal is to make the blog homepage feel like a timeline of progress. Visitors should be able to see how the projects are improving, what ideas keep coming back, and what kind of creative technology style kai9987kai is building.

Next improvement:

Add real screenshots, project links, GitHub buttons, and short “before vs after” sections to each post so every article becomes portfolio evidence.
