AI tool experiments that feel like creative instruments.

A creative AI dev-log about designing prompt tools, assistants and web interfaces that feel playful, useful, visual and portfolio-ready instead of plain text boxes.

✍️ Kai Piper 📅 09 Jun 2026 ⏱️ 5 min read 🤖 AI Tools

Build a tool that turns rough ideas into clearer concepts, options, previews and next steps.

AI tools can be more than chat boxes. They can behave like creative instruments: something you adjust, test, remix, compare and use to shape ideas faster. The best tools do not just answer a question. They help you explore possibilities.

For kai9987kai, this is a strong direction because it combines creative coding, interface design, prompt thinking and experimental web development. The goal is to make AI feel less hidden and more interactive.

A good AI interface should not feel like waiting for magic. It should feel like steering a creative machine.

The idea

A basic AI tool usually has one input box and one output area. That works, but it can feel flat. The user types something, waits, reads the answer, and then has to guess what to do next.

A better tool can guide the process. It can offer modes, sliders, examples, live previews, saved variations, comparison cards, prompt history, export buttons and clear next actions. The interface becomes part of the creative process instead of just a wrapper around a model.

Design goal:

Turn AI from a plain response generator into a visual, adjustable and reusable creative workflow that helps people shape ideas with more control.

Better interfaces

The interface should make the tool understandable before the user even types anything. Good labels, example prompts, mode buttons and short helper text can make the whole page feel more confident.

Instead of one giant text area, an AI tool can be split into useful parts: the raw idea, the style, the output type, the constraints, the preview and the saved results. That gives the user more control without making the tool feel complicated.

Modes Switch between brainstorm, improve, explain, summarise, generate, compare or polish.
Controls Use simple sliders or buttons for tone, detail, creativity, format and length.
Previews Show drafts, variants, cards, diagrams or structured outputs instead of one wall of text.
Exports Let the user copy, download, save, share or turn output into another format.

Feedback loops

AI tools become more useful when they support iteration. A first answer is rarely the final answer. The interface should make it easy to refine, regenerate, compare and build on a result without losing the previous version.

That could mean a version history, side-by-side comparison, “make it simpler” buttons, “more visual” buttons, “turn into HTML” buttons, or a small checklist that tells the user what could be improved next.

  • Regenerate safely: keep old outputs visible instead of replacing everything instantly.
  • Compare variants: show multiple directions so the user can choose the best one.
  • Explain changes: show what was improved, shortened, expanded or rewritten.
  • Use clear states: loading, success, error and empty states should all feel designed.
  • Make outputs reusable: copy buttons, downloads and templates make the tool more practical.

Next version

The next version should push the AI tools toward a proper creative dashboard. Instead of separate one-off pages, the site could have a small collection of AI instruments for writing, visual ideas, project planning, README drafting, UI improvement and prototype generation.

Good upgrades would include saved projects, prompt presets, output cards, dark/light themes, example galleries, better loading animations, export options and a “what changed” panel that explains how the AI transformed the input.

Portfolio angle:

This project can show prompt design, UX thinking, frontend engineering, creative tool design, accessibility, visual polish and the ability to turn AI into practical web apps.
